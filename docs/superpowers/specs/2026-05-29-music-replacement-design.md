# Replacement Background Music — CoinTex (all 7 music tracks)

**Date:** 2026-05-29
**Status:** Approved design (pending spec review)

## Problem

CoinTex ships 7 background-music WAVs in `music/`: the menu track
`bg_music_piano_flute.wav` and one per world `bg_music_world1.wav` …
`bg_music_world6.wav` (worlds: Meadow, Desert, Ocean, Cavern, Volcano, Space).
We are replacing all 7 with new, higher-quality, melodic, seamless-looping
tracks generated by a numpy synth tool — mirroring the workflow just used in the
sibling Gate Runner project, whose output the user approved.

The sound effects and the `music/originals/` folder are **kept as-is**.

## Decisions (confirmed with user)

- **Approach:** Port the proven Gate Runner synth tool (`tools/gen_world_music.py`)
  into CoinTex as a new **dev/build-time** module. numpy is the only dependency
  and is already present in CoinTex's venv and `requirements-dev.txt`; it is NOT
  a runtime dependency (`audio.py` only loads WAVs — its own comment notes "no
  numpy needed"). The 7 WAVs ship pre-generated under `music/`.
- **Vibe per track** (CoinTex is a cheerful kids' coin game):
  - **Menu (`bg_music_piano_flute.wav`) → Warm piano + flute melody**: gentle,
    pretty, welcoming — honors the existing piano/flute identity.
  - **W1 Meadow → Cheerful pastoral bounce** (~104 BPM): bright major-key,
    happy hopping rhythm.
  - **W2 Desert → Exotic groove** (~100 BPM): warm mid-tempo, harmonic-minor
    flavor, hand-percussion feel.
  - **W3 Ocean → Bright flowing aquatic** (~96 BPM): smooth flowing melody,
    bubbly arpeggio, soft pad, gentle sway — sparkling underwater/seaside.
  - **W4 Cavern → Mysterious echoey adventure** (~88 BPM): spacious slightly
    mysterious melody through heavy delay/reverb over a low pad — sense of
    wonder, exploring deep caves.
  - **W5 Volcano → Heroic action march** (~128 BPM): driving beat, stacked
    detuned-saw synth-brass triumphant melody, power-chord pad.
  - **W6 Space → Dreamy synthwave** (~92 BPM): warm detuned-saw pad, shimmering
    arpeggio through delay, smooth round bass, gentle backbeat.
- **Format:** 44.1 kHz, 16-bit, **mono**, ~20 s seamless loop per track
  (matches the existing tracks' sample rate/bit depth). ≈ 1.7 MB/file → ≈ 12 MB
  total (up from ~6 MB).

## Architecture

New module **`tools/gen_world_music.py`** in CoinTex, ported from Gate Runner's
reviewed implementation. It writes the 7 files into `music/`:
`bg_music_piano_flute.wav`, `bg_music_world1.wav` … `bg_music_world6.wav`,
overwriting the existing tracks.

Filenames are unchanged, so **no runtime code changes**: `audio.py` loads
`MENU_MUSIC` and `world_music_name(world)` with `loop=True` and picks up the new
files automatically.

### Synth core (reused from Gate Runner)

Band-limited oscillators (sine/triangle/saw/soft-square — no aliasing buzz),
ADSR, one-pole low-pass, feedback delay, synth drums (kick/snare/hat), a
beat-grid sequencer, an equal-power loop crossfade, normalize + tanh soft-clip
master, and per-track deterministic seeding in `main()` (so `--only <subset>`
rebuilds reproduce byte-for-byte).

### New voices (the only genuinely new synthesis)

- `_piano_voice(f, n)`: struck tone — a few decaying harmonics (1, 2, 3, …)
  with a fast attack and exponential decay; bright but warm.
- `_flute_voice(f, n)`: near-sine with a soft attack, gentle vibrato (small
  low-frequency pitch/amp modulation) and a touch of breath (low-level filtered
  noise) for an airy timbre.

These drive the menu builder (piano lead melody answered by flute, soft pad
underneath). The six world builders reuse the existing voice helpers
(`_pad`, `_bass_voice`, `_pluck_voice`, `_bell_voice`, `_brass_voice`,
`_tri_voice`, `_drum_track`, `_layered_pad`); Ocean and Cavern lean on
`_bell_voice`/`_tri_voice` through `delay` for flow/echo.

### Per-track builders + driver

`TRACKS` registry maps the exact existing filenames to builders. `main()`
(argparse `--out`/`--only`, per-track seed, crossfade + soft-clip, logs
peak/loopΔ/size) renders each track to a 44.1 kHz mono 16-bit WAV.

### Seamless looping

Bar-aligned loops with an equal-power tail→head crossfade so they repeat
click-free under Kivy's `loop=True`. The driver logs the loop-boundary
discontinuity per track.

## Out of scope

- The boss track — CoinTex has none.
- All SFX (`sfx_*.wav`, `coin.wav`, `char_death_flaute.wav`,
  `level_completed_flaute.wav`) — unchanged.
- The `music/originals/` folder — unchanged.
- Any `audio.py` / game / build (`buildozer.spec`) changes.
- Stereo output.

## Verification

- Headless test `tools/test_gen_world_music.py` (ported): synth-core unit tests
  + a structural test that builds all 7 tracks and asserts 44.1 k/mono,
  ~20 s (15–26 s), finite, peak ≤ 1.0 (no clipping), peak > 0.3 (not silent),
  loop seam < 0.08.
- Determinism: a full run reproduces the committed WAVs byte-for-byte, and an
  `--only` subset matches the full run for those tracks.
- Format check: each written WAV is 44.1 k / mono / 16-bit / ~20 s.
- Kivy-loader smoke test: `SoundLoader.load` opens all 7 (the real runtime path)
  with `loop=True` and no errors.
- Confirm SFX files and `music/originals/` are untouched (git status shows only
  the 7 `bg_music_*.wav` changed, plus the new tool/test/spec files).
- The user auditions audibly (`venv/bin/python main.py` on audio hardware) for
  final subjective quality — the tests cannot judge "sounds good."
